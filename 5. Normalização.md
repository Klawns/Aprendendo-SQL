# 1. Normalização

> A normalização é um conjunto de regras utilizado para organizar as tabelas de um banco de dados.

- Reduz redundância de dados.
- Aumenta a integridade das informações.
- Facilita a manutenção e a consistência do banco de dados.
- Existem seis formas normais, porém as três primeiras são as mais utilizadas.
- Uma tabela só pode avançar para a próxima forma normal quando atender às regras da anterior.

---

# 2. Primeira Forma Normal (1FN)

> A 1FN garante que cada tabela possua uma estrutura organizada e sem informações repetidas ou compostas.

### Regras

- Toda tabela deve possuir uma `PRIMARY KEY`.
- Todas as colunas devem ser atômicas.
    - Valores monovalorados.
    - Valores indivisíveis.
    - Atributos compostos devem ser separados em colunas.
    - Atributos multivalorados devem ser armazenados em outra tabela.

---

## 2.1 Atributos Compostos

> Um atributo composto é um campo que pode ser dividido em atributos menores.

- Não devem existir na 1FN.
- Devem ser separados em colunas individuais.
- Cada coluna deve armazenar apenas uma informação.

## 2.2 Atributos Multivalorados

> Um atributo multivalorado é um campo que pode armazenar mais de um valor para um mesmo registro.

- Não devem existir na 1FN.
- Cada valor deve ser armazenado em uma linha própria.
- Deve ser criada uma nova tabela relacionada à tabela principal.
