# 1. Introdução
Agora vamos sair do conjunto de DDL, agora vamos para o DML e DQL

- DML é o conjunto para manipular dados
- DQL é o conjunto para consultar dados

# 2. INSERT  
  
```sql  
INSERT INTO tabela (coluna1, coluna2, coluna3)  
VALUES (valor1, valor2, valor3);  
```  
  
- Insere um novo registro na tabela.  
- A ordem das colunas pode ser qualquer uma.  
- Os valores devem seguir exatamente a mesma ordem das colunas.  
- Colunas com `DEFAULT` ou auto incremento podem ser omitidas.  
  
Exemplo:  
  
```sql  
INSERT INTO todos (title, completed)  
VALUES ('Estudar SQL', false);  
```  
  
---  
  
# 3. SELECT  
  
```sql  
SELECT colunas  
FROM tabela;  
```  
  
- Consulta informações armazenadas em uma tabela.  
  
Buscar todas as colunas:  
  
```sql  
SELECT *  
FROM todos;  
```  
  
Buscar apenas algumas colunas:  
  
```sql  
SELECT id, title  
FROM todos;  
```  
  
Buscar apenas uma coluna:  
  
```sql  
SELECT title  
FROM todos;  
```  
  
---  
  
# 4. WHERE  
  
```sql  
SELECT colunas  
FROM tabela  
WHERE condição;  
```  
  
- Filtra os registros retornados pelo `SELECT`.  
- Sem o `WHERE`, todas as linhas da tabela são retornadas.  
  
## Operadores de comparação  
  
| Operador | Significado    |
| -------- | -------------- |
| =        | Igual          |
| != ou <> | Diferente      |
| >        | Maior          |
| <        | Menor          |
| >=       | Maior ou igual |
| <=       | Menor ou igual |
  
Exemplos:  
  
```sql  
SELECT *  
FROM todos  
WHERE id = 3;  
```  
  
```sql  
SELECT *  
FROM todos  
WHERE completed = true;  
```  
  
```sql  
SELECT *  
FROM todos  
WHERE id > 5;  
```  
  
---  
  
## Operadores Lógicos  
  
### AND  
  
Todas as condições devem ser verdadeiras.  
  
```sql  
SELECT *  
FROM todos  
WHERE completed = true  
AND id > 5;  
```  
  
---  
  
### OR  
  
Pelo menos uma condição deve ser verdadeira.  
  
```sql  
SELECT *  
FROM todos  
WHERE id = 1  
OR id = 5;  
```  
  
---  
  
### NOT  
  
Inverte uma condição.  
  
```sql  
SELECT *  
FROM todos  
WHERE NOT completed = true;  
```  
  
Equivale a:  
  
```sql  
SELECT *  
FROM todos  
WHERE completed = false;  
```  
  
---  
  
## BETWEEN  
  
Verifica se um valor está dentro de um intervalo.  
  
```sql  
SELECT *  
FROM todos  
WHERE id BETWEEN 5 AND 10;  
```  
  
Equivale a:  
  
```sql  
WHERE id >= 5  
AND id <= 10;  
```  
  
Também funciona com datas.  
  
```sql  
SELECT *  
FROM pedidos  
WHERE data BETWEEN '2025-01-01' AND '2025-12-31';  
```  
  
---  
  
## IN  
  
Verifica se um valor pertence a uma lista.  
  
Ao invés de:  
  
```sql  
WHERE id = 1  
OR id = 3  
OR id = 8;  
```  
  
Podemos fazer:  
  
```sql  
SELECT *  
FROM todos  
WHERE id IN (1, 3, 8);  
```  
  
Também funciona com textos.  
  
```sql  
SELECT *  
FROM usuarios  
WHERE cidade IN ('São Paulo', 'Rio de Janeiro');  
```  
  
---  
  
## LIKE  
  
Procura textos seguindo um padrão.  
  
### %  
  
Representa qualquer quantidade de caracteres.  
  
Contém:  
  
```sql  
WHERE title LIKE '%SQL';  
```  
  
Começa com:  
  
```sql  
WHERE title LIKE 'SQL%';  
```  
  
Termina com:  
  
```sql  
WHERE title LIKE '%SQL';  
```  
  
### _  
  
Representa exatamente um caractere.  
  
```sql  
WHERE nome LIKE 'J_o';  
```  
  
Resumo:  
  
- `%` → qualquer quantidade de caracteres.  
- `_` → exatamente um caractere.

# 5. GROUP BY

> Agrupa registros que possuem o mesmo valor em uma ou mais colunas.

- É uma cláusula da DQL.
    
- Trabalha em conjunto com as funções de agregação.
    
- Cada grupo gera um único resultado para a função de agregação.
    
- É executado após o `WHERE`.
    
- As colunas utilizadas no `GROUP BY` também devem aparecer no `SELECT` (salvo exceções específicas do SGBD).
### 5.1 Sintaxe

```sql
SELECT
    <coluna>,
    <função_agregação>
FROM <tabela>
WHERE <filtro>
GROUP BY <coluna>
ORDER BY <coluna>;
```

Exemplo:

```sql
SELECT
    departamento,
    COUNT(*) AS total_funcionarios
FROM funcionario
GROUP BY departamento;
```

# 6. ORDER BY  
  
```sql  
SELECT *  
FROM todos  
ORDER BY title ASC;  
```  
  
- Ordena os resultados retornados pelo `SELECT`.  
  
Tipos de ordenação:  
  
- `ASC` → Crescente (padrão).  
- `DESC` → Decrescente.  
  
Exemplos:  
  
```sql  
SELECT *  
FROM todos  
ORDER BY id ASC;  
```  
  
```sql  
SELECT *  
FROM todos  
ORDER BY id DESC;  
```  
  
```sql  
SELECT *  
FROM todos  
ORDER BY title ASC;  
```  
  
---  
# 6. UPDATE  
  
```sql  
UPDATE tabela  
SET coluna = valor  
WHERE condição;  
```  
  
- Atualiza um ou mais registros da tabela.  
- O `WHERE` define quais registros serão alterados.  
- Se o `WHERE` for omitido, **todos os registros da tabela serão atualizados**.  
  
Atualizando apenas uma coluna:  
  
```sql  
UPDATE todos  
SET completed = true  
WHERE id = 3;  
```  
  
Atualizando várias colunas:  
  
```sql  
UPDATE todos  
SET  
title = 'Estudar SQL',  
completed = false  
WHERE id = 5;  
```  
  
---  
  
# 7. DELETE  
  
```sql  
DELETE FROM tabela  
WHERE condição;  
```  
  
- Remove registros da tabela.  
- O `WHERE` define quais registros serão removidos.  
- Se o `WHERE` for omitido, **todos os registros da tabela serão apagados**.  
  
Removendo um registro:  
  
```sql  
DELETE FROM todos  
WHERE id = 3;  
```  
  
Removendo vários registros:  
  
```sql  
DELETE FROM todos  
WHERE completed = true;  
```  
  
Removendo todos os registros da tabela:  
  
```sql  
DELETE FROM todos;  
```

# 8. Alias (`AS`)

> Alias é um apelido temporário atribuído a colunas ou tabelas durante uma consulta.

- Torna a consulta mais legível.
    
- Facilita a escrita de consultas com `JOIN`.
    
- Existe apenas durante a execução da consulta.

## 8.1 Alias em colunas

```sql
SELECT SUM(valor) AS total_vendas
FROM pedidos;
```

## 8.2 Alias em tabelas

```sql
SELECT
    u.nome,
    p.id_pedido,
    p.valor
FROM usuarios AS u
JOIN pedidos AS p
    ON u.id_usuario = p.id_usuario
WHERE u.id_usuario = 15;
```

# 9. Funções de Agregação

> Funções de agregação realizam cálculos sobre um conjunto de registros e retornam um único valor.

- São utilizadas principalmente para gerar indicadores e estatísticas.
    
- Podem ser utilizadas juntamente com a cláusula `WHERE` para aplicar filtros antes do cálculo.
    
- Geralmente operam sobre colunas numéricas, exceto `COUNT`, que pode contar qualquer tipo de dado.
    
- São frequentemente utilizadas com `GROUP BY`.
    

## `COUNT`

> Retorna a quantidade de registros.

- `COUNT(*)` conta todas as linhas da consulta.
    
- `COUNT(coluna)` conta apenas os valores não nulos da coluna.
    
- `COUNT(DISTINCT coluna)` conta apenas valores distintos.
    
Exemplos:

```sql
SELECT COUNT(*) FROM aluno;
```

```sql
SELECT COUNT(nome) FROM aluno;
```

```sql
SELECT COUNT(DISTINCT nome) FROM aluno;
```

---

## `MAX`

> Retorna o maior valor de uma coluna.

Exemplo:

```sql
SELECT MAX(salario) FROM funcionario;
```

---

## `MIN`

> Retorna o menor valor de uma coluna.

Exemplo:

```sql
SELECT MIN(salario) FROM funcionario;
```

---

## `AVG`

> Retorna a média dos valores de uma coluna.

Exemplo:

```sql
SELECT AVG(salario) FROM funcionario;
```

---

## `SUM`

> Retorna a soma dos valores de uma coluna.

Exemplo:

```sql
SELECT SUM(salario) FROM funcionario;
```

# 10. Funções Numéricas

> Funções numéricas realizam operações matemáticas e manipulação de valores numéricos.

- São muito utilizadas para arredondamentos, truncamentos e formatação da projeção dos dados.

## `CEIL`

> Retorna o menor número inteiro maior ou igual ao valor informado.

- Arredonda sempre para cima.
    
Exemplo:

```sql
SELECT CEIL(17.4) AS valor;
```

Resultado:

```text
18
```

---

## `FLOOR`

> Retorna o maior número inteiro menor ou igual ao valor informado.

- Arredonda sempre para baixo.
    
Exemplo:

```sql
SELECT FLOOR(17.4) AS valor;
```

Resultado:

```text
17
```

---

## `TRUNCATE`

> Remove as casas decimais sem realizar arredondamento.

- O segundo parâmetro indica a quantidade de casas decimais que será mantida.

Exemplo:

```sql
SELECT TRUNCATE(22.772, 1) AS valor;
```

Resultado:

```text
22.7
```

---

## `ROUND`

> Arredonda um valor para a quantidade de casas decimais informada.

- Se o próximo dígito for menor que `5`, arredonda para baixo.
    
- Se o próximo dígito for maior ou igual a `5`, arredonda para cima.
    
Exemplo:

```sql
SELECT ROUND(55.8721, 2) AS valor;
```

Resultado:

```text
55.87
```
