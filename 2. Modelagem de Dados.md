# 1. Modelagem de Dados

> A modelagem de dados busca representar situações do mundo real dentro de um sistema.

- Facilita o planejamento e a organização do banco de dados.
- Define como os dados serão armazenados e relacionados.

---

## 1.1 Entidade

> Representa um objeto do mundo real que precisa ser armazenado no banco de dados.

- Geralmente será transformada em uma tabela.

Exemplos:

- Aluno
- Livro
- Cliente
- Pedido

---

## 1.2 Atributo

> Característica que descreve uma entidade.

- Geralmente será transformado em uma coluna da tabela.

Exemplo:

Aluno

- Nome
- Matrícula
- DataNascimento
- Telefone

---

# 2. Modelos de Banco de Dados

> Os modelos representam diferentes etapas do planejamento e da implementação de um banco de dados.

---

## 2.1 Modelo Conceitual

> Representa o negócio de forma abstrata.

- Focado nas regras de negócio.
- Identifica entidades, atributos e relacionamentos.
- Não define implementação.
- Independente do SGBD.

## 2.5 Modelo Físico

> Implementa o projeto utilizando comandos SQL (DDL).

- Define a estrutura física do banco.
- Produz as instruções SQL para criação do banco de dados.

Exemplo:

```sql
CREATE TABLE aluno (
    sexo CHAR(1),
    idade INT,
    nome TEXT,
    curso TEXT
);
```
