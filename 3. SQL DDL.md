# 1. SQL DDL

> DDL (Data Definition Language) é o subconjunto da SQL utilizado para definir e alterar a estrutura do banco de dados.

- Cria bancos de dados e tabelas.
- Altera a estrutura de tabelas existentes.
- Remove bancos de dados, tabelas e colunas.

---

# 2. Tipos de Dados

> Definem o tipo de informação que cada coluna poderá armazenar.

## 2.1 Tipos de Texto

### CHAR

> Armazena textos com tamanho fixo.

- Ideal para dados com tamanho conhecido.

Exemplos:

- UF (SP, RJ)
- Sexo (M, F)

---

### VARCHAR

> Armazena textos com tamanho variável.

- Ideal para nomes, e-mails e endereços.

Exemplos:

- João da Silva
- joao@email.com

---

### TEXT

> Armazena textos longos.

Exemplos:

- Observações
- Descrições

---

## 2.2 Tipos Numéricos

### INT

> Armazena números inteiros.

Exemplos:

- Idade
- Identificador

---

### FLOAT

> Armazena números com casas decimais.

Exemplos:

- Salário
- Preço

---

## 2.3 Tipos de Data e Hora

### DATE

> Armazena apenas a data.

Exemplo:

- 2026-06-22

---

### TIME

> Armazena apenas a hora.

Exemplo:

- 14:30:00

---

### DATETIME

> Armazena data e hora.

Exemplo:

- 2026-06-22 14:30:00

---

## 2.4 CHAR x VARCHAR

### CHAR

> Possui tamanho fixo.

- Sempre ocupa o tamanho definido.
- Indicado para informações com tamanho conhecido.

Exemplo:

```text
CHAR(10)

J O R G E _ _ _ _ _
```

A palavra **JORGE** ocupa os 10 caracteres reservados.

---

### VARCHAR

> Possui tamanho variável.

- Ocupa apenas o espaço necessário.
- Indicado para informações com tamanho variável.

Exemplo:

```text
VARCHAR(10)

J O R G E
```

A palavra **JORGE** ocupa apenas os caracteres utilizados.

---

## 2.5 ENUM

> `ENUM` é um tipo de dado que limita os valores possíveis de uma coluna a uma lista pré-definida.

- Contribui para a **Integridade de Domínio**.
    
- Não faz parte do padrão SQL.
    
- Está disponível em alguns SGBDs, como MySQL e PostgreSQL (implementações diferentes).
    
Exemplo:
`status ENUM('PENDENTE', 'PAGO', 'CANCELADO')`
