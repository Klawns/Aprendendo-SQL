# 1. Constraints

> Constraints são regras aplicadas pelo banco de dados para garantir a integridade, consistência e validade das informações armazenadas.

Elas permitem definir regras como:

- Identificação única dos registros.
    
- Relacionamentos entre tabelas.
    
- Valores obrigatórios.
    
- Valores únicos.
    
- Validação dos dados.
    
- Valores padrões.
    

As constraints podem ser criadas durante o `CREATE TABLE` ou adicionadas posteriormente com `ALTER TABLE`.

---

# 2. Principais Constraints

## `PRIMARY KEY`

> Identifica cada registro de uma tabela de forma única.

- Garante a **Integridade de Entidade**.
    
- Não permite valores `NULL`.
    
- Não permite valores duplicados.
    
- Cada tabela possui apenas uma chave primária (podendo ser composta por várias colunas).
    
Exemplo:

`id INT PRIMARY KEY`

---

## `FOREIGN KEY`

> Cria um relacionamento entre tabelas.

- Garante a **Integridade Referencial**.
    
- Deve referenciar uma `PRIMARY KEY` ou uma coluna `UNIQUE`.
    
- Evita referências inválidas.
    
- Pode definir ações como `CASCADE`, `RESTRICT`, `SET NULL` e `SET DEFAULT`.
    
Exemplo:

```sql
FOREIGN KEY (cliente_id)
REFERENCES Cliente(id)
```

---

## `UNIQUE`

> Garante que os valores de uma coluna não sejam repetidos.

- Impede registros duplicados.
    
- Permite valores `NULL` (na maioria dos SGBDs).
    
- Normalmente cria um índice automaticamente.
    
Exemplo:

`email VARCHAR(150) UNIQUE`

---

## `NOT NULL`

> Torna obrigatório o preenchimento de uma coluna.

- Impede valores `NULL`.
    
- Contribui para a **Integridade de Domínio**.
    
- Não impede valores duplicados.
    
Exemplo:

`nome VARCHAR(100) NOT NULL`

---

## `CHECK`

> Valida uma condição antes que os dados sejam armazenados.

- Garante a **Integridade de Domínio**.
    
- Impede valores que não satisfaçam a condição definida.
    
Exemplo:

`idade INT CHECK (idade >= 18)`

---

## `DEFAULT`

> Define um valor padrão quando nenhum valor é informado durante a inserção.

- Evita valores ausentes em determinadas colunas.
    
- Pode ser utilizado junto com outras constraints.
    
Exemplo:

`status VARCHAR(20) DEFAULT 'PENDENTE'`

---
# 3. Nomeando Constraints

> É recomendado atribuir nomes às constraints para facilitar a manutenção, documentação e identificação de erros.

- Caso um nome não seja informado, o SGBD geralmente gera um automaticamente.

Exemplo:

```sql
CONSTRAINT PK_CLIENTE
PRIMARY KEY (id_cliente)
```

```sql
CONSTRAINT FK_PEDIDO_CLIENTE
FOREIGN KEY (id_cliente)
REFERENCES Cliente(id_cliente)
```

```sql
CONSTRAINT CK_IDADE
CHECK (idade >= 18)
```
